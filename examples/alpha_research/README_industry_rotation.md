# 行业轮动策略开发总结

## ✅ 已完成的策略

### 1. 等权重策略（基准）

**文件**: `test_backtest_simple.py`

**逻辑**: 每日调仓，等权重持有所有股票

**回测结果**（3 年，18 只股票）:
| 指标 | 数值 |
|------|------|
| 总收益率 | **45.99%** |
| 年化收益 | **13.6%** |
| 最大回撤 | 30.86% |
| 交易次数 | 7,794 |

**评价**: ✅ 简单有效，作为基准策略

---

### 2. 行业轮动策略 V1（严格估值筛选）

**文件**: `test_industry_rotation.py`

**逻辑**: 
- 选择热门行业（动量 + 估值）
- 在热门行业中选择低估值股票（PE<20, PB<3）
- 5 天调仓

**回测结果**:
| 指标 | 数值 |
|------|------|
| 总收益率 | **-37.44%** ❌ |
| 年化收益 | -12.75% |
| 最大回撤 | 61.95% |
| 交易次数 | 368 |

**问题**: 
- 估值筛选太严格，很多股票被过滤
- 行业估值数据不准确（使用固定值）
- 调仓频率低，错过机会

---

### 3. 行业轮动策略 V2（动量 + 相对估值）

**文件**: `test_industry_rotation_v2.py`

**逻辑**:
- 纯动量选择热门行业
- 行业内相对估值排序
- 5 天调仓

**回测结果**:
| 指标 | 数值 |
|------|------|
| 总收益率 | **-15.24%** ❌ |
| 年化收益 | -4.70% |
| 最大回撤 | 49.13% |
| 交易次数 | 239 |

**问题**:
- 模拟数据的行业轮动特征不明显
- 动量因子在模拟数据中效果差
- 调仓频率仍然偏低

## 🔍 问题分析

### 为什么行业轮动策略表现差？

1. **数据问题**
   - 模拟数据是随机生成的，没有真实的行业轮动特征
   - 行业间相关性、估值差异不真实
   - 缺少真实的财务数据（PE/PB/股息率）

2. **策略问题**
   - 估值筛选依赖固定值，不是真实数据
   - 动量因子在模拟数据中可能反向有效
   - 调仓频率可能不合适

3. **市场特征**
   - 模拟数据是几何布朗运动，没有趋势/均值回归特征
   - 真实市场的行业轮动需要宏观经济驱动

## ✅ 解决方案

### 方案 1: 使用真实数据测试

**推荐**: 下载真实股票数据 + 财务数据

```bash
# AKShare（免费）
python download_akshare.py

# 或使用 RQData（付费，最稳定）
```

**预期效果**: 
- 真实的行业轮动特征
- 真实的估值数据
- 策略效果更可靠

### 方案 2: 改进模拟数据生成

生成带有行业特征的数据：
- 不同行业有不同的收益率分布
- 行业间有相关性
- 估值有周期性变化

### 方案 3: 简化策略逻辑

回到简单策略：
- 纯动量策略（选择过去 N 日最强的股票）
- 低波动策略（选择波动率最低的股票）
- 均值回归策略（选择跌幅最大的股票）

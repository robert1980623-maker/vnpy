## 🌞 下午持仓下载（17:15）

### 配置

```bash
# 脚本
download_holdings.sh

# Cron 表达式
15 17 * * 1-5  (周一 - 周五 17:15)

# Job ID
holdings-download-20260304
```

### 特性

- ✅ **只下载持仓**: 从虚拟账户读取持仓列表（15-25 只）
- ✅ **快速完成**: 数据量小，5-10 分钟完成
- ✅ **重试机制**: 5 次重试，间隔 5 分钟
- ✅ **分段下载**: 每批 5 只，避免并发过高

### 数据用途

```
15:00 收盘 → 17:15 下载持仓 → 20:00 复盘
   ↓
计算当日盈亏
生成复盘报告
```

## 🎯 为什么需要下午持仓下载？

### 问题场景

**如果没有下午持仓下载**:

```
20:00 复盘
  ↓
读取持仓数据
  ↓
❌ 数据只到昨天（03-03）
  ↓
❌ 无法计算今日（03-04）盈亏
  ↓
❌ 复盘报告不准确
```

### 解决方案

**有下午持仓下载**:

```
17:15 下载持仓
  ↓
✅ 数据包含今日收盘价（03-04 15:00）
  ↓
20:00 复盘
  ↓
✅ 准确计算今日盈亏
✅ 准确的持仓表现分析
✅ 准确的复盘报告
```

## ⚠️ 注意事项

### 1. 无持仓情况

如果虚拟账户没有持仓：

```
⚠️  无持仓股票，跳过下载
```

**正常现象**，建仓后会自动下载。

### 2. 周末和节假日

- **凌晨下载**: 正常运行，但数据无更新（股市休市）
- **下午下载**: 仅在周一 - 周五执行（`* * 1-5`）

### 3. 数据验证

复盘前检查数据是否最新：

```bash
# 检查持仓股票数据日期
for symbol in $(cat accounts/virtual_2026_account.json | python3 -c "import json,sys; print(' '.join([p['symbol'] for p in json.load(sys.stdin)['positions']]))"); do
    echo -n "$symbol: "
    tail -1 data/akshare/bars/${symbol//./_}.csv | cut -d',' -f2
done
```

### 4. 日志清理

```bash
# 保留最近 7 天日志
find ./logs -name "*.log" -mtime +7 -delete
```

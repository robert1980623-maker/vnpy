**优化目标**: 对服务器友好 + 自动重试机制

## ✅ 已完成的优化

### 1. 禁用下午下载任务

**原任务**: 17:00 每日数据下载  
**状态**: ❌ 已禁用  
**原因**: 凌晨下载已足够，避免重复

### 2. 凌晨下载优化为增量模式

**任务时间**: 凌晨 01:00  
**脚本**: `batch_download_incremental.sh`  
**特性**:
- ✅ 增量下载：只下载没有的股票或更新已有股票
- ✅ 智能检测：检查文件是否存在、日期是否最新
- ✅ 跳过已更新：避免重复下载

### 3. 自动重试机制

**配置**:
- **最大重试次数**: 5 次
- **重试间隔**: 20 分钟（1200 秒）
- **触发条件**: 下载失败时自动重试

**流程**:
```
下载失败 → 等待 20 分钟 → 重试 1
        → 等待 20 分钟 → 重试 2
        → 等待 20 分钟 → 重试 3
        → 等待 20 分钟 → 重试 4
        → 等待 20 分钟 → 重试 5
        → 仍失败 → 记录日志，标记为失败
```

**总耗时**（最坏情况）:
- 5 次重试 × 20 分钟 = 100 分钟（1 小时 40 分钟）
- 加上初始下载，总共约 2 小时
- 完全在 01:00-09:00 的时间窗口内

## 🔧 技术实现

### batch_download_incremental.sh

**核心功能**:

1. **智能筛选**
   ```bash
   # 检查是否需要更新
   needs_update() {
       local file="${DATA_DIR}/${symbol//./_}.csv"
       
       # 文件不存在
       if [ ! -f "$file" ]; then
           return 0  # 需要下载
       fi
       
       # 检查日期
       local last_date=$(tail -1 "$file" | cut -d',' -f2)
       if [ "$last_date" != "$YESTERDAY" ] && [ "$last_date" != "$TODAY" ]; then
           return 0  # 需要更新
       fi
       
       return 1  # 跳过
   }
   ```

2. **重试机制**
   ```bash
   download_stock() {
       local retry_count=0
       
       while [ $retry_count -le $MAX_RETRIES ]; do
           # 下载
           python3 download_data_akshare.py --symbols ${symbol} ...
           
           if [ $? -eq 0 ]; then
               return 0  # 成功
           fi
           
           retry_count=$((retry_count + 1))
           
           # 等待 20 分钟后重试
           if [ $retry_count -le $MAX_RETRIES ]; then
               sleep $RETRY_INTERVAL  # 1200 秒
           fi
       done
       
       return 1  # 失败
   }
   ```

3. **日志记录**
   - 所有操作记录到 `logs/download_YYYYMMDD_HHMMSS.log`
   - 包含成功、失败、重试详情
   - 便于排查问题

## ⚠️ 注意事项

### 1. 首次运行

第一次运行时会下载所有股票，耗时较长（约 30-60 分钟）。

**解决**: 正常运行，后续会自动跳过已更新的股票。

### 2. 周末和节假日

股市休市，无新数据。

**处理**: 脚本会自动检测，如果数据已是最新，直接跳过。

### 3. 网络故障

如果连续 5 次重试都失败：

**处理**:
1. 检查日志文件：`logs/download_*.log`
2. 手动重试：`./batch_download_incremental.sh`
3. 检查网络：`ping api.example.com`

### 4. 磁盘空间

日志文件会累积。

**处理**: 定期清理旧日志：
```bash
# 保留最近 7 天的日志
find ./logs -name "download_*.log" -mtime +7 -delete
```

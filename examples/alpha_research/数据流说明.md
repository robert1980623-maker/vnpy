# 数据流说明 - 每日选股与数据下载

**核心原则**: 每天选股应该基于前一天收盘后下载的数据（真实数据）

---

## ✅ 正确的流程

### Day 1 (2026-03-03 周一)

```
09:00 股市开盘
15:00 股市收盘 → 产生 03-03 的完整日线数据
17:00 数据下载任务执行
    ↓
    调用：python3 download_data_akshare.py --start 20260303 --end 20260303
    ↓
    保存到：data/akshare/bars/*.csv
    ↓
    ✅ 现在数据文件包含 03-03 的收盘价
```

### Day 2 (2026-03-04 周二)

```
09:00 每日选股任务执行
    ↓
    读取：data/akshare/bars/*.csv (包含 03-03 数据)
    ↓
    选股逻辑：基于 03-03 的收盘价计算财务指标
    ↓
    生成：reports/stock_selection_2026-03-04.json
    ↓
    ✅ 选股结果基于真实的 03-03 收盘数据
```

## 📅 日期计算规则

| 任务 | 执行时间 | 使用的数据日期 | 说明 |
|------|---------|--------------|------|
| **数据下载** | Day N 17:00 | Day N (当日) | 下载当日收盘数据 |
| **每日选股** | Day N+1 09:00 | Day N (昨日) | 基于昨日数据选股 |
| **交易执行** | Day N+1 09:00 | - | 使用今日开盘价或昨日收盘价 |

## 🔧 下载脚本配置

### batch_download_enhanced.sh

```bash
# 下载昨天的数据（因为今天的数据要等到 17:00 收盘后才有）
YESTERDAY=$(date -v-1d +%Y%m%d)  # macOS
TODAY=$(date +%Y%m%d)

# 调用下载脚本
python3 download_data_akshare.py --max 30 --start ${YESTERDAY} --end ${TODAY}
```

### 下载范围

- **默认**: 最近 1 天（昨日）
- **原因**: 今日数据要等到 17:00 收盘后才完整
- **选股**: 使用昨日数据，今天 09:00 执行

## ⚠️ 常见问题

### 问题 1: 选股用的是旧数据

**原因**: 数据下载任务没有执行或失败

**解决**:
```bash
cd /Users/rowang/projects/vnpy/examples/alpha_research
./batch_download_enhanced.sh
```

### 问题 2: 数据日期不对

**检查**:
```bash
# 查看最新数据日期
tail -1 data/akshare/bars/*.csv | grep -v "^==" | cut -d',' -f2 | sort -u | tail -5
```

**期望**: 应该看到昨天的日期（如 2026-03-03）

### 问题 3: 周末和节假日

**规则**:
- 周六日股市休市，无新数据
- 周一 09:00 选股 → 使用周五的收盘数据
- 数据下载任务在休市日仍然执行，但 AKShare 返回空数据或重复数据

**解决**: 选股脚本应该检查数据日期，如果最新数据是 2 天前，发出警告。

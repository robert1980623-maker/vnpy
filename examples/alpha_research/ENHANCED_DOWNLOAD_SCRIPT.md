# 🚀 增强版分批下载脚本

**创建时间**: 2026-03-02 07:35

**状态**: ✅ 已就绪

---

## 📊 配置参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `BATCH_SIZE` | 5 | 每批下载数量 |
| `BATCH_DELAY` | 60 | 批次间隔（秒） |
| `MAX_RETRIES` | 2 | 最大重试次数 |
| `TOTAL_STOCKS` | 50 | 总下载数量 |

---

## ⚙️ 功能特性

### 1. 更多批次
- **之前**: 4 批 × 5 只 = 20 只
- **现在**: 10 批 × 5 只 = 50 只
- **可扩展**: 修改 `TOTAL_STOCKS` 即可

### 2. 更长间隔
- **之前**: 30 秒
- **现在**: 60 秒
- **优势**: 更稳妥，避免限流

### 3. 自动重试
- **重试次数**: 最多 2 次
- **重试间隔**: 10 秒
- **优势**: 网络波动自动恢复

### 4. 详细统计
- 成功/失败统计
- 重试次数统计
- 总耗时统计
- 数据文件验证

## 📈 时间估算

### 当前配置 (50 只)

```
批次数量：10 批
每批耗时：~30 秒 (下载) + 60 秒 (间隔) = 90 秒
总耗时：10 × 90 秒 ≈ 15 分钟
重试缓冲：+5 分钟 (如有重试)
总计：约 20 分钟
```

### 扩展配置 (100 只)

```bash
# 修改脚本中的 TOTAL_STOCKS=100
批次数量：20 批
总耗时：20 × 90 秒 ≈ 30 分钟
重试缓冲：+10 分钟
总计：约 40 分钟
```

### 极限配置 (300 只 - 沪深 300)

```bash
# 修改脚本中的 TOTAL_STOCKS=300
批次数量：60 批
总耗时：60 × 90 秒 ≈ 90 分钟
重试缓冲：+30 分钟
总计：约 2 小时
```

## 🎯 凌晨下载优化建议

### 推荐配置 (凌晨 1 点执行)

```bash
TOTAL_STOCKS=100    # 下载 100 只
BATCH_DELAY=60      # 间隔 60 秒
MAX_RETRIES=2       # 重试 2 次
```

**优势**:
- ✅ 时间充裕 (约 40 分钟)
- ✅ 覆盖更多股票
- ✅ 容错性好
- ✅ 不影响其他任务

### 激进配置 (凌晨 1 点执行)

```bash
TOTAL_STOCKS=300    # 下载 300 只 (沪深 300)
BATCH_DELAY=45      # 间隔 45 秒
MAX_RETRIES=3       # 重试 3 次
```

**优势**:
- ✅ 覆盖全部沪深 300
- ✅ 约 2 小时完成
- ✅ 凌晨 3 点前完成
- ✅ 数据更全面

**风险**:
- ⚠️ 时间较长
- ⚠️ 可能影响其他任务
